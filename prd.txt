# UpDown 게임 요구사항 정의서 (PRD)

## 1. 게임 개요
- UpDown 게임은 컴퓨터가 1에서 100 사이의 임의의 숫자를 생각하고, 사용자가 이를 맞추는 게임입니다.
- 단일 페이지 애플리케이션(SPA)으로 HTML, CSS, JavaScript를 사용하여 개발합니다.

## 2. 웹 접근성 (A11y) 요구사항
- **스크린 리더 지원**: 숫자를 입력하고 결과를 확인할 때, 판정 결과(UP, DOWN, 정답, 실패 등)를 스크린 리더가 즉시 소리 내어 읽을 수 있도록 `aria-live` 속성을 적절히 활용합니다.
- **키보드 접근성**: 마우스 없이 `Tab` 키를 이용해 모든 대화형 요소(입력란, 버튼 등)에 접근할 수 있어야 하며, 숫자를 입력하고 `Enter` 키를 눌러 바로 제출할 수 있어야 합니다. 명확하고 시인성 높은 `:focus-visible` 스타일을 적용합니다.
- **화면 보조 기능**:
  - 폰트 크기 확대/축소 기능 제공.
  - 고대비 테마 전환 기능 제공.
- **청각 피드백**: 판정 결과에 따른 효과음을 제공하며, 소리를 켜고 끌 수 있는 음소거 버튼을 제공합니다.

## 3. 디자인 및 사용자 경험 (UX)
- **프리미엄 다크 모드**: 세련되고 눈이 편안한 어두운 테마 기반의 디자인을 적용합니다.
- **실시간 비주얼 피드백**:
  - **UP**: 따뜻한 오렌지/레드 계열의 배경 애니메이션 및 상승 화살표 표시.
  - **DOWN**: 시원한 블루/사이언 계열의 배경 애니메이션 및 하강 화살표 표시.
  - **성공(정답)**: 불꽃놀이(Confetti) 애니메이션 효과와 축하 팝업.
- **범위 가이드**: 입력한 결과에 따라 다음 시도에서 입력해야 할 범위(예: "현재 범위: 25 ~ 50")를 화면 및 프로그레스 바 형태로 실시간 시각화합니다.
- **시도 기록(History)**: 이전 시도 기록(시도 번호, 입력한 숫자, 판정 결과)을 직관적인 타임라인 또는 카드 형태로 화면에 누적하여 보여줍니다.

## 4. 게임 시스템 기능
- **게임 모드**:
  - **연습 모드**: 횟수 제한 없이 자유롭게 정답을 맞추는 모드.
  - **도전 모드**: 제한된 기회(예: 7회) 안에 정답을 맞춰야 하는 모드.
- **기록 저장 (High Score)**: 브라우저의 `localStorage`를 활용하여 도전 모드 성공 시 가장 적은 시도 횟수를 최기 기록으로 보관하고 메인 화면에 노출합니다.
